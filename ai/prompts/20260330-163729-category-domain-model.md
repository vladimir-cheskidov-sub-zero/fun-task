# Prompt log

## Summary
Create the `Category` domain model and supporting value objects with strict invariants.

## User prompt
Создай доменную модель Category и VO, требующиеся для отражения корректного состояния доменной модели.
Ограничения:
* Модель должна быть Rich моделью в терминологии DDD
* Модель не должна позволять создавать себя в невалидном состоянии
* Модель не должна иметь методов-setter-ов. Вместо этого должны быть методы выполняющие бизнес действия
* Методы модели не должны позволять привести модель в невалидное состояние
* Конструктор доменной модели должен содержать всю логику инициализации объекта. Инициализация не должна быть размазана по статическим фабрикам
* Должна быть проведена обработка ошибок - исключения должны быть доменными, лежать в слое домена и говорить о нарушении инвариантов модели или о невозможности выполнить бизнесоперацию.
* VO должны быть иммутабельными и не позволять создавать себя в невалидном состоянии.
* Сигнатуры всех публичных методов доменной модели должно отражать четкий контракт и не иметь nullable параметров или параметров со значению по умолчанию
